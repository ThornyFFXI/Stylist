XML Layout

 

<stylist> <!-- All files must start with stylist node. -->

 

            <settings> <!-- Settings is a top-level node that lets you configure stylist’s behavior.--> 

                        <!-- All setting nodes are optional.  If they are not specified, or an invalid value is used, the default value(listed in white here) will be used. -->

                       <noblinkself>true</noblinkself> <!-- When enabled, your character will never blink except when applying filters or using /sl blink. -->

<noblinktarget>true</noblinktarget> <!-- When enabled, your current target will never blink except when applying filters or using /sl blink. -->

<noblinkothers>false</noblinkothers> <!-- When enabled, no players will blink except when applying filters or using /sl blink. -->

            </settings>

 

            <player name=”Thorny”> <!-- You may have any number of player nodes to customize player appearances. -->

                        <race>mithra</race>

                        <body>Tartarus Platemail</body> <!--Each player node may have any amount of slot-specific filters.  These follow the same rules as when adding filters by typed commands.-->

                        <filter item=”Sagitta” slot=”main”>Spharai</filter> <!--You may also insert model specific filters inside a player node, they will only apply to that player.-->

            </player>

 

            <filter item=”Kraken Club” slot=”main”>Heartbeater</filter>

            <filter item=”Kraken Club” slot=”sub”>Heartbeater</filter> <!--Filters that are not inside a player node will apply to everyone.  You can have any amount of them.-->

 

</stylist>
